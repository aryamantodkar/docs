## Overview

The Workspace API allows you to create and manage workspaces, configure organizations, manage team members, and control workspace settings like scheduling and enabled AI providers.

## Endpoints

### create

Create a new workspace with a new organization.

**Type**: Mutation (Protected)

**Rate Limit**: 3 requests per hour

<ParamField path="organizationName" type="string" optional>
  Name of the organization (2-80 characters). Defaults to workspace name if not provided.
</ParamField>

<ParamField path="name" type="string" required>
  Name of the workspace (2-50 characters)
</ParamField>

<ParamField path="slug" type="string" required>
  URL-friendly slug for the workspace (2-50 characters)
</ParamField>

<ParamField path="domain" type="string" required>
  Domain to track (2-50 characters)
</ParamField>

<ResponseField name="workspace" type="object">
  The created workspace object
</ResponseField>

<ResponseField name="org" type="object">
  The created organization object
</ResponseField>

<ResponseField name="isFirstWorkspace" type="boolean">
  Whether this is the user's first workspace
</ResponseField>

<CodeGroup>
```typescript Request
const result = await trpc.workspace.create.mutate({
  name: "Acme Corp Tracking",
  slug: "acme-corp",
  domain: "acme.com",
  organizationName: "Acme Corporation"
});
```

```typescript Response
{
  workspace: {
    id: "ws_abc123",
    name: "Acme Corp Tracking",
    slug: "acme-corp",
    domain: "acme.com",
    tenantId: "org_xyz789",
    schedule: null,
    enabledProviders: "[\"openai\",\"anthropic\",\"google\"]",
    createdAt: Date,
    updatedAt: Date
  },
  org: {
    id: "org_xyz789",
    name: "Acme Corporation",
    slug: "acme-corp"
  },
  isFirstWorkspace: true
}
```
</CodeGroup>

### setSchedule

Set or update the cron schedule for automated prompt collection. When a schedule is set, an immediate agent run is triggered.

**Type**: Mutation (Authorized Workspace)

<ParamField path="workspaceId" type="string" required>
  ID of the workspace
</ParamField>

<ParamField path="schedule" type="string | null" required>
  Cron expression for the schedule (e.g., "0 0 * * *" for daily at midnight), or null to disable
</ParamField>

<ResponseField name="workspace" type="object">
  Updated workspace object
</ResponseField>

<ResponseField name="immediateRun" type="object">
  Status of the immediate agent run that was triggered
  
  - `status: "queued"` - Run was successfully queued with jobId
  - `status: "empty"` - No prompts to analyze
  - `status: "failed"` - Run failed with error
  - `status: "not-requested"` - Schedule was set to null (disabled)
</ResponseField>

<CodeGroup>
```typescript Request
const result = await trpc.workspace.setSchedule.mutate({
  workspaceId: "ws_abc123",
  schedule: "0 0 * * *" // Daily at midnight
});
```

```typescript Response
{
  workspace: {
    id: "ws_abc123",
    schedule: "0 0 * * *",
    updatedAt: Date
  },
  immediateRun: {
    status: "queued",
    jobId: "job_xyz789"
  }
}
```
</CodeGroup>
